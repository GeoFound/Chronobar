# Chronobar AI Assistant 产品契约 v1.0

## 1. 文档定位

本文档定义 Chronobar AI Assistant 的用户可见产品契约，包括能力范围、拒答范围、来源显示、不确定性表达、动作分级、确认点与审计提示。

本文档解决的是“用户如何理解这个助手能做什么、不能做什么、答案依据什么、下一步是否可执行”，而不是运行时实现细节。

## 2. 产品目标

Chronobar AI Assistant 的目标不是把聊天做得像人，而是把 AI 做成：

- 可控
- 透明
- 可引用
- 可拒答
- 可审计

在金融敏感场景下，高质量不等于“都回答”，而是“在证据充分、边界明确时可靠回答，在高风险和高不确定性场景下稳定拒答或要求确认”。

## 3. 能力范围

AI Assistant 当前推荐面向以下能力范围：

### 3.1 产品问答

可回答：
- Chronobar 功能说明
- 协议文档解释
- 配置字段解释
- UI 功能入口说明
- 常见错误码与故障排查线索

### 3.2 回测解读

可回答：
- 收益、回撤、胜率、滑点、异常区间解释
- 回测结果阅读建议
- 指标含义说明
- 需要人工进一步核验的可疑点提示

### 3.3 风控解释

可回答：
- 命中哪条规则
- 为什么被拦截或告警
- 建议如何修复配置、参数或流程

### 3.4 概念解释

可回答：
- 术语解释
- 指标说明
- 交易概念说明
- Chronobar 内部对象和模块关系说明

### 3.5 研究辅助

可回答：
- 研究计划草稿
- 复盘提纲
- 配置草案
- 代码骨架草案

约束：
- 输出仅作为建议或草稿，不直接落地执行。

## 4. 禁止范围

AI Assistant 默认不得：

- 提供投资建议或收益承诺
- 代替用户做交易决策
- 自动执行下单、撤单、改仓位等动作
- 在无证据时对系统状态作出确定性判断
- 绕过风控与权限模型访问核心能力
- 伪装外部来源或伪造引用

## 5. 用户界面最小契约

AI Assistant 的聊天面板、侧边 Copilot 或页内 Copilot 至少应显示以下信息：

- **能力边界**：可回答产品、回测、风控、概念问题；不提供投资建议
- **来源标签**：系统数据 / Chronobar 文档 / 受控外部来源 / 模型推断
- **动作级别**：解释型 / 建议型 / 修改型 / 执行型
- **不确定性提示**：确定 / 高概率 / 需要更多上下文 / 无法确认
- **下一步提示**：可继续追问、去查看文档、去打开页面、需人工确认

## 6. 来源显示契约

每个回答都必须尽可能显示其主要依据来源。

### 6.1 来源类别

- **系统数据**：来自回测结果、风险事件、配置状态、工作区状态等结构化内部数据
- **Chronobar 文档**：来自 README、协议文档、FAQ、帮助文档、错误说明
- **受控外部来源**：来自白名单外部文档、规则库或说明页面
- **模型推断**：无法直接引用文档或数据、仅依据一般性推理给出的说明

### 6.2 展示要求

- 当回答基于系统数据时，应优先显示系统数据来源。
- 当回答基于文档时，应显示文档标题或链接。
- 当回答混合多类来源时，应显式标记。
- 当只有模型推断、没有稳定证据时，应降低确定性并提示用户谨慎使用。

## 7. 不确定性表达契约

AI Assistant 必须对不确定性显式表达，不允许所有回答都呈现同等自信程度。

建议采用四级表达：

- **确定**：证据明确且结构化一致
- **高概率**：证据较充分，但存在少量推断
- **需要更多上下文**：当前信息不足，需补充页面、对象或参数
- **无法确认**：缺乏足够证据，或超出能力边界

## 8. 动作分级契约

### 8.1 L0 解释型

示例：
- 解释协议字段
- 解释回测指标
- 解释风控拦截原因

处理规则：
- 可直接回答
- 不需要额外确认

### 8.2 L1 建议型

示例：
- 给出排查建议
- 给出回测阅读建议
- 给出参数观察建议

处理规则：
- 可直接返回
- 必须显式声明为建议而非结论

### 8.3 L2 修改型

示例：
- 生成配置草稿
- 生成代码草稿
- 生成参数调整建议

处理规则：
- 允许生成草稿
- 必须经过用户确认才能写入或应用

### 8.4 L3 执行型

示例：
- 下单
- 撤单
- 修改仓位
- 修改风控开关并立即生效

处理规则：
- 默认禁用
- 不属于 AI Assistant 当前可直接执行范围

## 9. 拒答契约

AI Assistant 必须在以下场景稳定拒答，而不是勉强给出模糊回答：

- 明确要求投资建议或收益承诺
- 请求代替用户作出交易决定
- 请求直接执行交易动作
- 请求绕过风控、权限或审计机制
- 缺少足够证据但用户要求确定性结论
- 超出 Chronobar 产品范围或未授权数据范围

拒答时应：
- 明确说明拒答原因
- 尽量提供可接受的替代路径
- 引导用户查看文档、切换页面、补充上下文或人工确认

## 10. 确认点契约

以下操作必须进入确认流程：

- 写配置文件
- 生成并落地代码
- 修改参数并准备应用
- 启用外部搜索或外部上下文
- 批量导出可能含敏感内容的结果

确认时至少应显示：
- 即将执行的动作
- 影响范围
- 来源依据
- 是否可撤销

## 11. 页内 Copilot 建议落点

Chronobar 最适合优先落地 AI Assistant 的页面包括：

- **帮助/文档页**：产品问答、术语解释
- **回测页**：回测解读、异常区间说明
- **风控页**：规则命中解释、修复建议
- **配置页**：字段解释、Schema 说明、配置草稿建议

不建议第一阶段优先把它做成“全站万能聊天框”。

## 12. 审计提示契约

涉及系统数据、外部来源、修改建议或拒答的回答，应向用户明确提示：

- 是否记录本次请求
- 是否使用了系统数据
- 是否访问了外部来源
- 是否涉及需确认动作

## 13. 成功衡量标准

产品侧至少应持续跟踪：

- 产品问答满意度
- 回测解读引用率
- 风控解释准确率
- 拒答准确率
- 误导率
- 用户对来源显示的理解度
- 修改建议的确认转化率
