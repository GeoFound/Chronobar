# Chronobar AI Assistant 产品契约 v1.0

## 1. 文档定位

本文档定义 Chronobar AI Assistant 的用户可见产品契约，包括能力范围、拒答范围、来源显示、不确定性表达、动作分级、确认点与审计提示。
本文档约束的是 Chronobar 产品内部面向产品使用者的 **产品 AI**，不定义仓库演进过程中用于文档、代码、测试、维护与迁移的 **仓库 AI**。

本文档解决的是“用户如何理解这个助手能做什么、不能做什么、答案依据什么、下一步是否可执行”，而不是运行时实现细节。

## 2. 产品目标

Chronobar AI Assistant 的目标不是把聊天做得像人，而是把 AI 做成：

- 可控
- 透明
- 可引用
- 可落实
- 可拒答
- 可审计

在金融敏感场景下，高质量不等于“都回答”，而是“在证据充分、边界明确时可靠回答，在高风险和高不确定性场景下稳定拒答或要求确认”。

它既不是代替用户判断的交易代理，也不是只会吐出一次性草稿的聊天玩具，而是帮助用户把自己的研究意图、配置想法、策略表达和说明内容落实为可验证、可保存、可继续编辑的用户资产。
这里的“用户”指 Chronobar 产品使用者，而不是仓库维护者；仓库维护者与仓库 AI 的协作边界不由本契约定义。

在 Chronobar 中，AI Assistant 的定位还包括：帮助用户在稳定平台骨架之上装配自己的系统，而不是替平台重新定义骨架本身。用户决定策略、指标、工作区、连接方式与研究流程，平台负责稳定边界，AI 负责在这些边界内帮助用户加速构建。

## 3. 能力范围

AI Assistant 当前推荐面向以下能力范围：

### 3.1 产品问答

可回答：
- Chronobar 功能说明
- 协议文档解释
- 配置字段解释
- UI 功能入口说明
- 常见错误码与故障排查线索

### 3.2 回测解读

可回答：
- 收益、回撤、胜率、滑点、异常区间解释
- 回测结果阅读建议
- 指标含义说明
- 需要人工进一步核验的可疑点提示

### 3.3 风控解释

可回答：
- 命中哪条规则
- 为什么被拦截或告警
- 建议如何修复配置、参数或流程

### 3.4 概念解释

可回答：
- 术语解释
- 指标说明
- 交易概念说明
- Chronobar 内部对象和模块关系说明

### 3.5 研究辅助

可回答：
- 研究计划草稿
- 复盘提纲
- 配置候选产物
- 代码候选产物
- 指标、策略与工作区候选产物
- Gateway 接入说明、能力声明与配置候选产物
- 因子假设、特征工程方案与实验配置候选产物
- 实验对比解读与研究发布前检查清单
- 用户说明文案候选产物

约束：
- 输出可以是可用候选产物，而不只是一次性聊天草稿。
- 候选产物应尽量达到最小可用门槛，例如 schema 可校验、结构可加载、代码可运行或可继续回测验证。
- 写入正式对象、应用到工作区或替换现有文件前，必须经过用户确认。
- 涉及 Gateway、账户接入或连接资产时，可以生成配置候选产物和能力说明，但不得静默保存口令、认证码或其他敏感凭据。
- 研究类输出应优先形成可复跑实验配置、可比较结果或候选发布记录，而不是伪装成已确认结论。
- 产物默认归用户所有，不得默认外送、共享或作为未声明训练材料使用。

### 3.6 真实世界研究辅助

可回答：
- 交易所规则说明
- 合约与品种公开资料说明
- 受控外部公告、规则页和说明页检索
- 与当前问题直接相关的受控外部上下文整合

约束：
- 外部访问必须受白名单和策略控制。
- 必须显式标记为受控外部来源，而不是伪装成 Chronobar 内部事实。
- 应尽可能保留来源标题、链接、抓取时间或版本信息。

### 3.7 记忆协作

可回答：
- 记住用户常用术语、页面偏好和工作流偏好
- 在同一工作区内延续最近的研究上下文
- 在用户授权范围内复用用户确认过的偏好设定

约束：
- 记忆必须可管理、可删除、可说明用途。
- 默认不把记忆视为事实真相源，仍需结合文档、系统数据或外部来源。
- 未经用户授权，不得把策略细节、交易偏好或敏感资产静默写入长期记忆。

### 3.8 用户系统装配辅助

Chronobar AI Assistant 可以辅助用户装配自己的系统，至少包括：

- 策略、指标、参数与工作区候选产物生成
- Gateway 接入说明、能力差异解释和候选配置生成
- 本地研究流程、复盘模板和用户说明资产整理
- 用户确认后资产之间的引用关系梳理

约束：
- AI 只能帮助用户在平台骨架上构建，不得擅自修改核心协议边界、风控总边界或权限模型。
- AI 生成的内容应先进入构建态，而不是默认直接进入运行态。
- 只有经过用户确认并完成最小校验的正式对象，才允许进入运行态或替换现有对象。

### 3.9 多因子研究辅助

Chronobar AI Assistant 可以辅助用户完成多因子研究工作流，至少包括：

- 因子定义草稿、标签定义草稿与实验配置候选产物
- 因子表解读、实验对比总结与研究工作区整理
- 研究候选对象的发布前检查、影响范围说明与确认建议

约束：
- 研究结果必须区分“草稿实验 / 已复跑结果 / 待发布对象 / 已发布对象”。
- AI 不得把未复跑、未确认的研究结果表述为可直接运行的正式结论。
- 研究态产物进入运行态前，必须显示发布目标、依赖资产、最小校验结果和确认回执。

## 4. 禁止范围

AI Assistant 默认不得：

- 提供投资建议或收益承诺
- 代替用户做交易决策
- 自动执行下单、撤单、改仓位等动作
- 在无证据时对系统状态作出确定性判断
- 绕过风控与权限模型访问核心能力
- 伪装外部来源或伪造引用

## 5. 用户界面最小契约

AI Assistant 的聊天面板、侧边 Copilot 或页内 Copilot 至少应显示以下信息：

- **能力边界**：可回答产品、回测、风控、概念问题；不提供投资建议
- **来源标签**：系统数据 / Chronobar 文档 / 受控外部来源 / 模型推断
- **动作级别**：解释型 / 建议型 / 修改型 / 执行型
- **产物状态**：仅回答 / 候选产物 / 待确认变更 / 已应用回执
- **研究状态**：草稿实验 / 已复跑 / 待发布 / 已发布
- **不确定性提示**：确定 / 高概率 / 需要更多上下文 / 无法确认
- **记忆提示**：是否使用了会话记忆或用户记忆
- **下一步提示**：可继续追问、去查看文档、去打开页面、需人工确认

## 6. 来源显示契约

每个回答都必须尽可能显示其主要依据来源。

### 6.1 来源类别

- **系统数据**：来自回测结果、风险事件、配置状态、工作区状态等结构化内部数据
- **Chronobar 文档**：来自 README、协议文档、FAQ、帮助文档、错误说明
- **受控外部来源**：来自白名单外部文档、规则库或说明页面
- **模型推断**：无法直接引用文档或数据、仅依据一般性推理给出的说明

### 6.2 展示要求

- 当回答基于系统数据时，应优先显示系统数据来源。
- 当回答基于文档时，应显示文档标题或链接。
- 当回答混合多类来源时，应显式标记。
- 当回答基于受控外部来源时，应尽量显示来源标题、域名和时间信息。
- 当只有模型推断、没有稳定证据时，应降低确定性并提示用户谨慎使用。

## 7. 不确定性表达契约

AI Assistant 必须对不确定性显式表达，不允许所有回答都呈现同等自信程度。

建议采用四级表达：

- **确定**：证据明确且结构化一致
- **高概率**：证据较充分，但存在少量推断
- **需要更多上下文**：当前信息不足，需补充页面、对象或参数
- **无法确认**：缺乏足够证据，或超出能力边界

## 8. 动作分级契约

### 8.1 L0 解释型

示例：
- 解释协议字段
- 解释回测指标
- 解释风控拦截原因

处理规则：
- 可直接回答
- 不需要额外确认

### 8.2 L1 建议型

示例：
- 给出排查建议
- 给出回测阅读建议
- 给出参数观察建议

处理规则：
- 可直接返回
- 必须显式声明为建议而非结论

### 8.3 L2 修改型

示例：
- 生成配置候选产物
- 生成代码候选产物
- 生成参数调整建议
- 生成工作区或说明文案候选产物

处理规则：
- 允许生成候选正式产物
- 必须显示影响范围、主要变更点、来源依据和最小校验结果
- 必须经过用户确认才能写入、应用或替换现有对象

### 8.4 L3 执行型

示例：
- 下单
- 撤单
- 修改仓位
- 修改风控开关并立即生效

处理规则：
- 默认禁用
- 不属于 AI Assistant 当前可直接执行范围

## 9. 拒答契约

AI Assistant 必须在以下场景稳定拒答，而不是勉强给出模糊回答：

- 明确要求投资建议或收益承诺
- 请求代替用户作出交易决定
- 请求直接执行交易动作
- 请求绕过风控、权限或审计机制
- 缺少足够证据但用户要求确定性结论
- 超出 Chronobar 产品范围或未授权数据范围

拒答时应：
- 明确说明拒答原因
- 尽量提供可接受的替代路径
- 引导用户查看文档、切换页面、补充上下文或人工确认

## 10. 确认点契约

以下操作必须进入确认流程：

- 写配置文件
- 生成并落地代码
- 修改参数并准备应用
- 将构建态候选产物发布为运行态正式对象
- 发布因子、实验或研究候选对象
- 启用外部搜索或外部上下文
- 批量导出可能含敏感内容的结果

确认时至少应显示：
- 即将执行的动作
- 影响范围
- 来源依据
- 最小校验结果
- 是否可撤销

## 11. 记忆使用契约

涉及记忆时，AI Assistant 至少应向用户明确：

- 使用的是会话记忆还是用户记忆
- 该记忆是否来自用户明确授权
- 用户是否可以删除或清空这段记忆
- 该记忆是否被用于生成候选产物或个性化答案

## 12. 页内 Copilot 建议落点

Chronobar 最适合优先落地 AI Assistant 的页面包括：

- **帮助/文档页**：产品问答、术语解释
- **回测页**：回测解读、异常区间说明
- **风控页**：规则命中解释、修复建议
- **配置页**：字段解释、Schema 说明、配置候选产物与确认应用
- **策略/指标页**：参数说明、候选产物生成、验证结果回显
- **研究/因子页**：因子表解读、实验对比、研究工作区整理与发布确认

不建议第一阶段优先把它做成“全站万能聊天框”。

## 13. 审计提示契约

涉及系统数据、外部来源、修改建议或拒答的回答，应向用户明确提示：

- 是否记录本次请求
- 是否使用了系统数据
- 是否访问了外部来源
- 是否使用了记忆
- 是否涉及需确认动作

## 14. 成功衡量标准

产品侧至少应持续跟踪：

- 产品问答满意度
- 回测解读引用率
- 风控解释准确率
- 拒答准确率
- 误导率
- 用户对来源显示的理解度
- 修改建议的确认转化率
- 候选产物的采用率
- 研究配置复跑成功率
- 研究候选对象的发布确认转化率
- 外部来源引用后的人工纠错率
- 记忆命中后的用户纠正率
