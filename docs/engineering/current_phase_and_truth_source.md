# Chronobar 当前阶段与真相源说明 v1.0

## 1. 文档定位

本文档定义 Chronobar 当前正式阶段、阶段声明的真相源优先级、实验性运行时代码的解释方式，以及进入下一阶段前必须满足的收口条件。

它是 `README.md`、`docs/roadmap.md`、`ai-instructions.md`、`docs/CONTRIBUTING.md` 与 `docs/engineering/delivery_master_plan.md` 的统一锚点，用于防止“仓库已有部分实现”与“阶段 gate 已通过”被混为一谈。

## 2. 当前正式阶段

Chronobar 当前正式阶段为 **M1 基线收敛**。

在 [`delivery_master_plan.md`](delivery_master_plan.md) 的执行分层中，对应 **P0：M1 基线冻结**。

说明：

- 仓库中已经存在少量 `core/`、`examples/`、`tests/integration/` 下的实验性运行时代码、演示脚本和预演产物。
- 这些内容可以作为后续实现参考，但**不构成 P1 / M2 已通过 gate** 的依据。
- 在 P1 gate 未被正式验收前，任何文档都不得将当前仓库状态描述为“P1 核心闭环完成”。

## 3. 阶段真相源优先级

当多个文档对当前阶段表述不一致时，按以下优先级解释：

1. 本文档：[`current_phase_and_truth_source.md`](current_phase_and_truth_source.md)
2. 路线图：[`../roadmap.md`](../roadmap.md)
3. AI/代理协作约束：[`../../ai-instructions.md`](../../ai-instructions.md) 与仓库根目录 `AGENTS.md`
4. 工程门槛与阶段 gate：[`engineering_baseline.md`](engineering_baseline.md)、[`delivery_master_plan.md`](delivery_master_plan.md)
5. README、贡献指南、快速开始等导航型文档

治理规则：

- `README.md` 负责对外展示与导航，不单独发明阶段状态。
- `tests/` 中的基线测试必须和本文件保持一致，而不是反向覆盖本文件。
- 若未来阶段推进到 P1 / M2，本文件必须与 `docs/roadmap.md`、`CHANGELOG.md` 同步更新。

## 4. 当前范围冻结

在 M1 阶段，Chronobar 的正式范围冻结如下：

- **单用户**：默认面向单一桌面用户，不讨论团队协作或云托管多租户。
- **单机**：默认本地单节点运行，不引入分布式协调和远程服务依赖。
- **单 Python sidecar**：正式架构仍为单核心进程，不在 M1 推进多进程拆分。
- **单账户优先**：M2-M4 的正式运行范围默认只承诺单账户链路。
- **单活跃策略运行时**：早期正式链路只承诺单活跃策略执行；多策略仅允许离线回测对比或研究场景讨论。
- **单一规范数据口径**：实时输入、落盘、查询、回放必须共享同一套标准对象与主数据路径。

若某项工作会突破以上边界，必须先更新本文件、`docs/roadmap.md` 与相应 gate 文档，再讨论实现。

## 5. M1 阶段允许与禁止事项

### 5.1 允许事项

- 收敛协议、配置、Schema、测试、CI 和工程基线
- 补齐运行前设计缺口，例如范围冻结、非功能约束、数据生命周期与恢复路径
- 保留少量实验性代码或演示脚本，用于验证后续实现方向
- 为 M2 任务包准备样例、基准、目录约定和契约样例

### 5.2 禁止事项

- 以实验性代码存在为由，直接宣称 P1 / M2 已完成
- 在没有正式 gate 验收的前提下，把 README、测试或对外表述切换到“核心闭环完成”
- 把多账户、多策略并发、云同步、复杂恢复编排提前做成默认范围
- 让临时样例、探索脚本、预演实现反向定义协议边界

## 6. 进入下一阶段前必须收口的事项

在允许进入 P1 / M2 正式实现前，至少需要满足以下条件：

- 当前阶段口径已统一，导航文档与测试不再冲突
- 当前范围冻结已明确记录并被执行文档引用
- 存储生命周期、保留策略、备份/恢复与损坏处理路径已成文
- 非功能基线、性能目标、基准测法与最小可观测性要求已成文
- `just docs-check` 可通过；若本地环境具备依赖，`just check` 也应作为进入实现前的门禁

## 7. 维护规则

- 当阶段状态、范围边界、gate 条件发生变化时，必须同步更新：
  - `README.md`
  - `docs/roadmap.md`
  - `ai-instructions.md`
  - `docs/CONTRIBUTING.md`
  - `CHANGELOG.md`
- 任何新出现的“阶段完成总结”“预演总结”“试验记录”都必须明确标注其是否具备正式放行效力。
- 若仓库出现阶段口径冲突，优先修复文档和测试，不带着歧义继续扩展实现范围。
