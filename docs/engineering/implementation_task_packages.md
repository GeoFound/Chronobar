# Chronobar 实施任务包 v1.0

## 1. 文档定位

本文档把 [`delivery_master_plan.md`](delivery_master_plan.md) 拆成可交给任务 AI 或开发者逐个执行的任务包。

每个任务包都必须有：

- 明确目标
- 依赖输入
- 产出文件或模块
- 验证方法
- 阻塞条件
- 进入下一任务的前提

## 2. 使用规则

- 一次只推进一个任务包到 `in_progress`
- 每个任务包结束前必须先做验证，再进入下一个任务包
- 若验证失败且涉及协议边界，应回退到上一阶段 gate，而不是硬继续写代码

## 3. P1：M2 核心闭环任务包

### P1-T1 仓库运行时代码初始化

- **目标**：创建最小 Python runtime 目录骨架
- **依赖**：M1 gate 完成
- **产出**：`app/`、`core/`、`rules/`、`compute/`、`storage/`、`ui/desktop/`、`tests/unit/`、`tests/integration/`
- **验证**：目录结构与 `engineering_baseline.md` 一致；`pytest` 可发现新增测试目录
- **阻塞条件**：目录设计与现有协议边界冲突

### P1-T2 标准对象与异常层

- **目标**：把协议中的最小对象映射到运行时代码
- **依赖**：P1-T1
- **产出**：核心模型、枚举、异常定义
- **验证**：对象可实例化；协议测试通过；mypy 不报公共接口错误
- **阻塞条件**：需要改协议字段才能实现

### P1-T3 EventEngine

- **目标**：实现统一事件总线
- **依赖**：P1-T2
- **产出**：事件注册、订阅、分发、trace_id 保留、异常隔离
- **验证**：事件投递单测与集成测试通过
- **阻塞条件**：事件模型与 `event_protocol.md` 不一致

### P1-T4 RuleEngine 最小 session 判定

- **目标**：实现夜盘与 trading_date 最小规则
- **依赖**：P1-T2
- **产出**：交易时段模板加载、session 判定
- **验证**：夜盘样例测试通过
- **阻塞条件**：规则样例与协议描述冲突

### P1-T5 BarAggregator + 1 个指标

- **目标**：完成 Tick -> Bar 与最小指标计算
- **依赖**：P1-T3、P1-T4
- **产出**：Bar 聚合器、至少 1 个指标
- **验证**：1 分钟 Bar 聚合测试通过
- **阻塞条件**：计算结果不稳定或不可复验

### P1-T6 UiBridge 最小投影

- **目标**：向前端提供最小 Query / Subscription 投影
- **依赖**：P1-T3、P1-T5
- **产出**：最小桥接接口与标准返回结构
- **验证**：契约测试与样例结构一致
- **阻塞条件**：桥接字段与协议样例偏离

### P1-T7 核心闭环演示

- **目标**：把前面任务串成可运行闭环
- **依赖**：P1-T1 ~ P1-T6
- **产出**：开发演示脚本 / README / 集成测试
- **验证**：Tick -> Event -> Bar -> Indicator -> Snapshot 全链路可复现
- **进入下一阶段前提**：P1 gate 全部通过

## 9. 任务 AI 执行约束

交给任务 AI 执行时，必须遵守：

- 不得跨阶段抢跑
- 不得在 gate 未通过时开始下一阶段
- 不得用临时字段绕过协议
- 不得以“先写出来再整理”为理由污染边界
- 不得默认开启高风险 AI 能力
- 所有新增模块都必须附最小测试与验证命令

## 10. 任务包完成定义

一个任务包只有在以下条件全部满足时才算完成：

- 代码或文档产出已入仓
- 对应测试通过
- README / 契约 / 样例已同步（如适用）
- 风险与限制已记录
- 下一任务包所需输入已经稳定
